## 3. 提示順序ルール

- 上記のペア番号順は提示順ではない。実装時にランダムシャッフルすること
- **制約**: 連続する5問以内に同一ニーズが3回以上出現しないこと
- シャッフル後に制約を検証し、違反があれば再シャッフル（最大100回試行）
- シャッフルに使用したシード値を保存し、再現可能にすること

## 4. 回答データの出力スキーマ

```json
{
  "respondent_id": "uuid-v4",
  "timestamp": "ISO8601",
  "shuffle_seed": 12345,
  "responses": [
    {
      "pair_index": 1,
      "need_a": "ability_utilization",
      "need_b": "achievement",
      "winner": "ability_utilization"
    },
    ...
  ]
}
```

- `responses` は必ず210件
- `pair_index` は上記ペア番号（1–210）に対応
- `winner` は `need_a` または `need_b` の値と一致すること
